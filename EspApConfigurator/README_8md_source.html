3 ## Summary
4 
5 This is a re-implementation and modification of the functionality available in the [WiFiManager library](https://github.com/tzapu/WiFiManager).
6 
7 Differences:
8 
9 - Less flash memory usage (faster to upload your code)
10 - Everything done with timeslices an Mutila Modes, making it possible to run your own code while also being in AP mode (WiFiManager blocks)
11 - Extensible - add your own configuration options which save to EEPROM

75 ### Mode 1: Single page interface
76 
77 Typically for projects where web server only runs in ModeAP. No password protection for changing settings, but ModeAP should have password set.
78

86 - Routes:
87  - onNotFound(handleNotFound);
88  - on("/", handleSinglePage);
89  - on("/save", handleSingleSave);
90  - on("/wifi", handleSingleCancel);
91  - on("/r", handleRescan);

120 ## Pre-requisites
121 
122 * Install the Arduino IDE 1.6.6 or later
123  * All OSes: [download link](https://www.arduino.cc/en/Main/Software)
124  * Major Linux distros carry it in their software repositories, or the link above may be used
125 * In the Arduino IDE, install ESP8266 board support from Tools -> Board -> Boards Manager
126 * In the Arduino IDE, install ESP8266 tools from Sketch -> Include Library -> Manage Libraries
127 * Install the [Mutila](https://github.com/matthewg42/Mutila) library
128 
129 ## Building Using the Arduino IDE
130 
131 This method is preferred on Windows and for casual users who just want to build and use the project.
132 
133 * Open Arduino IDE
134 * Open firmware/firmware.ino
135 * Set the board you are using (under the Tools -> Board menu)
136 * Connect board with USB cable
137 * Click the Upload button on the tool bar
138 
139 ## Using make on Linux
140 
141 This alternative build system may also be used. It is an alternative to using the Arduino IDE and may be preferred by some users.
142 
143 * Install the esp/Arduino makefile: https://github.com/plerup/makeEspArduino
144 * Edit the esp.mk file and set the include line to point at where you installed makeEspArduino.mk (or set the ESP_MAKEFILE environment variable)
145 * Edit firmware/Makefile and uncomment a BOARD for the board you're using (to see a list of possible boards, run "make list_boards" from the firmware directory)
146 * Connect your ESP board with a USB cable
147 * Press and hold the flash button on the board and use the command "make flash" from the firmware directory
