52 ## Version 1.1.0
53 
54 This release is primarily to address the *Millis wrap problem*, but I ended up bundling some other changes in there too. A few classes have changed, which could potentially break compatibility with existing sketches, but only for classes which are probably only used internally, like RawButton which doesn't make a lot of sense to use in an end-user sketch.
55 
56 ### Features
57 - Implement MillisSince and DoEvery in Millis.h to address Millis wrap problem.
58 - Use new functions throughout Mutila code and examples.
59 - Improved documentation.
60 - Move to using stdint.h types rather than simple int and friends, for more consistent behaviour across platforms.
61 - New ModeManager class and example.
62 - Changed a lot of #defined globals into static const variables in appropriate classes to clean up the global name-space; removed Mutila.h.
63 - Created and documented standard test rig using a Nano, DFPlayer Mini, potential divider, two buttons and a sonic ranger module.
64 - Rearranged button hierarchy. New AbstractButton and AbstractDebouncedButton classes help reduce duplicated code. Three basic immediate button types (which don't need a time slice):
65  - Renamed RawButton to DigitalInputButton.
66  - Created Analog ButtonInputButton.
67  - DualInputButton, which can be any combination of DigitalInputButton and ButtonInputButton, and can work in Or, Xor and And modes.
68 
69 ### Bug fixes
70 - Heartbeat::Mode On mode now doesn't continuously switch LED on and off every update().
71 - Fix Mode::update() with interval set calling too many times immediately after Mode startup.
